# 项目概述
- 这是一个数据库内核开发项目，正在一步一步开发数据库内核。
- 使用C++17和cmake进行项目开发
- 当前处于迭代一阶段，各模块实现都是精简的，只完成最基础的功能
- 源码文件为cc后缀，而不是cpp

# 构建
- 工程目录下执行sh build.sh会构建生成libnebuladb.so
- 测试目录下执行sh build.sh会生成对应的可执行文件
# 关键约定
- 我会在doc一个一个模块的写实现原理，然后你帮我实现，如果实现的模块代码中依赖未开发的模块的接口，你需要在未开发的模块中写一个函数，直接返回。
- 实现原理文档中的遗留任务、易错点不用实现，只是我作为后续优化的标记，也不需要发给大模型。
- 类名使用驼峰规则，其它使用C++ 谷歌风格
- 不能破坏我的项目架构
- 数据库内核模块目录下有include/存放的是对外接口的头文件，源码和模块内部头文件直接放在模块目录下，例如src/common/include/下放对外接口的头文件, src/common/下放cc文件
- 模块的ut测试用例放在test/dt/ut/模块名/目录下，测试用例命名约定ut_模块名.cc
- 每次让你实现某个模块功能时，你需要补充对应的测试用例到指定的目录（即上一条约定），没让你实现的时候，不要写代码
